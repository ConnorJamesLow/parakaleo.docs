Overview

Version 1 Milestones

The following are a list of features (minor versions) that must be implemented to consider this major version of Parakaleo “complete”:

  1. Initial Counseling Resources features:
  • Create, Edit, View, and Delete Counseling Resources.
  • Table View of Resources
  1. Counseling Resource Tags
  2. Counseling Resource search API:
  • Search by Author, Title, or Subject
  • Tag and Date filters
  1. Login module
  • Basic single-factor authentication.
  • JWT auth system for API and front-end application.

Versioning

As of v1.0.0-alpha2, each project will have its own version independent of the other projects. Version criteria also varies between projects.

REACT APP

Versioning for ParakaleoClient (React App) uses the milestones to determine major versions, and individual features as minor versions. Bug fixes and feature improvements are patches.

v<version-milestone>.<milestone-feature>.<patch>

API & CORE

These projects follow semantic versioning.

Architectural Plan

Project Structure

Parakaleo is divided into three parts: the front end react app, the RESTful service API, and a Core library that provides some consistent functions between apps.

REACT APP

A Razor web application that relies on React for interfaces.

API

ParakaleoAPI is a REST service for Parakaleo applications.

Dependencies: ParakaleoCore

CORE

ParakaleoCore contains external libraries (e.g. MongoDB) and provides custom interfaces for dependant applications, like ParakaleoAPI.

Dependencies: Newtonsoft.JSON, MongoDB.Driver
