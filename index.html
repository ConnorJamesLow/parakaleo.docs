Overview

Version 1 Milestones

The following are a list of features (minor versions) that must be implemented to consider this major version of Parakaleo “complete”:

  1. Initial Counseling Resources features:
  • Create, Edit, View, and Delete Counseling Resources.
  • Table View of Resources
  1. Counseling Resource Tags
  2. Counseling Resource search API:
  • Search by Author, Title, or Subject
  • Tag and Date filters
  1. Login module
  • Basic single-factor authentication.
  • JWT auth system for API and front-end application.

Versioning

As of v1.0.0-alpha2, each project will have its own version independent of the other projects. Version criteria also varies between projects.

REACT APP

Versioning for ParakaleoClient (React App) uses the milestones to determine major versions, and individual features as minor versions. Bug fixes and feature improvements are patches.

v<version-milestone>.<milestone-feature>.<patch>

API & CORE

These projects follow semantic versioning.

Architectural Plan

Project Structure

Parakaleo is divided into three parts: the front end react app, the RESTful service API, and a Core library that provides some consistent functions between apps.

REACT APP

A Razor web application that relies on React for interfaces.

API

ParakaleoAPI is a REST service for Parakaleo applications.

Dependencies: ParakaleoCore

CORE

ParakaleoCore contains external libraries (e.g. MongoDB) and provides custom interfaces for dependant applications, like ParakaleoAPI.

Dependencies: Newtonsoft.JSON, MongoDB.Driver

Application State Design

Application state (the current snapshot of the UI data--its objects and variables used to determine what renders) is managed by a Redux store, the one and only source of truth for the application.

REDUCERS

In general, a reducer is a function that takes two values and outputs a single value; typical structure for a reducer looks like this: `(initialState, someValue) => reducedValue`. Redux uses reducers to make changes to the stored application state. Read more about reducers in general and for Redux.

The reducer structure looks like this (initial state is listed above the line, with action handlers below the line).

SELECTORS

Selectors are functions used to filter data from a store. The first parameter given to a selector is always the store/application state.

ACTIONS

To modify the store via the reducers, we expose actions: functions that return objects used by reducers to determine how to modify data. A typical action will return an object with this structure: `value => ({ type: ACTION_TYPE, payload: value })`. Actions are passed to React Components as props in the `connect()` method and must be used to make changes to the application state. When an action function is called (ie., when an action is dispatched), the connected store intercepts and parses it within the reducer.

ACTION TYPES

Every action has a type. These are used by reducers to determine what to do with an action dispatch.
